## 2. Enable Payment Methods

Stripe → **Settings → Payment methods**

### 2a. SEPA Direct Debit
- Toggle **SEPA Direct Debit** → **On**
- Mandate: Auto-generated by Stripe Payment Element — no custom mandate needed
- Delay: funds arrive T+5 business days (set booking as "pending" until settled)

### 2b. iDEAL (Netherlands)
- Toggle **iDEAL** → **On**
- Currencies accepted: EUR only
- Redirect: iDEAL redirects to bank — `return_url` in `confirmPayment()` handles it
- Note: Stripe auto-converts completed iDEAL to SEPA mandate for recurring

### 2c. Bancontact (Belgium)
- Toggle **Bancontact** → **On**
- Currencies accepted: EUR only
- Redirect-based (same as iDEAL)

### 2d. BLIK (Poland)
- Toggle **BLIK** → **On**
- Currencies accepted: PLN only
- In-page 6-digit code entry — no redirect; `redirect: 'if_required'` handles this

### 2e. Klarna
- Toggle **Klarna** → **On**
- Markets to enable: DE, AT, GB, FR, SE, NO, DK, FI, NL, BE, IT, ES
- Requires Klarna business terms acceptance in Dashboard (one-time)
- Klarna adds 4–6% fee on top of Stripe processing — update `GATEWAY_CONFIG` if adjusting pricing

### 2f. Apple Pay
- Toggle **Apple Pay** → **On**
- Add domain: `api.utubooking.com` + `www.utubooking.com`
- Download domain verification file → host at `/.well-known/apple-developer-merchantid-domain-association`
- Currencies: EUR + GBP wallets (add both under Wallet settings)

### 2g. Google Pay
- Toggle **Google Pay** → **On**
- No domain verification required
- Currencies: EUR + GBP

### 2h. Link (Stripe's express checkout — optional)
- Toggle **Link** → **On** if you want one-click checkout for returning Stripe users
- Adds `link` to GB method order in `PaymentRouter.ts`

## 3. Webhook Configuration

Stripe → **Developers → Webhooks → Add endpoint**

**Endpoint URL**: `https://api.utubooking.com/api/payments/stripe/webhook`

**Events to subscribe** (add all):
```
payment_intent.succeeded
payment_intent.payment_failed
payment_intent.processing
payment_intent.canceled
payment_intent.requires_action
charge.dispute.created
charge.refunded
```

After saving, copy the **Signing secret** (`whsec_...`) → set in all environments:
```bash
# backend/.env (local dev)
STRIPE_WEBHOOK_SECRET=whsec_XXXXXXXXXXXXXXXXXXXXXXXX

# AWS SSM (prod) — run in each region
aws ssm put-parameter --name /utubooking/prod/STRIPE_WEBHOOK_SECRET \
  --value "whsec_XXXXXXXXXXXXXXXXXXXXXXXX" \
  --type SecureString --overwrite
```

## 4. API Keys

Stripe → **Developers → API keys**

Copy the **Publishable key** (pk_live_...) and **Secret key** (sk_live_...).

```bash
# frontend/.env.local

# backend/.env
```

> **Never** commit live keys. Store in AWS Secrets Manager and inject at deploy time.

## 6. TWINT (Switzerland) — Separate from Stripe

> Note: TWINT is NOT configured in Stripe Dashboard. It uses the TWINT Partner API directly.
> However, Stripe now offers native TWINT for CHF (2024+) as an alternative — see below.

### Option A: TWINT Partner API (current implementation)
1. Apply for TWINT partnership at https://developer.twint.ch
2. Submit business documentation (takes 2–4 weeks)
3. Obtain `CLIENT_ID`, `CLIENT_SECRET`, `PARTNER_ID`
4. Set env vars:
   ```bash
   TWINT_ENVIRONMENT=production
   TWINT_CLIENT_ID=your_client_id
   TWINT_CLIENT_SECRET=your_client_secret
   TWINT_PARTNER_ID=your_partner_id
   TWINT_WEBHOOK_SECRET=your_webhook_signing_secret
   TWINT_WEBHOOK_URL=https://api.utubooking.com/api/payments/twint/webhook
   ```
5. Register webhook URL with TWINT Partner Portal

### Option B: Stripe-native TWINT (simpler, higher fee)
If TWINT Partner API approval is delayed:
1. Stripe Dashboard → Payment methods → Switzerland → Enable **TWINT**
2. Change `SwitzerlandPaymentSelector` to delegate to `EuropePaymentSelector` with `currency='CHF'`
3. Stripe Payment Element will auto-show TWINT QR for CHF
4. No separate TWINT contract needed — but fees are Stripe rate (~1.4% + €0.25) + TWINT surcharge

## 9. Run the DB Migration

Migration `20260324000028_add_eu_payment_methods` adds `sepa_debit`, `ideal`, `bancontact`, `klarna`, `blik`, `stripe_element`, `twint`, `apple_pay`, `google_pay` to the `payment_method` enum.

**Run on all shard DBs** (from repo root):

```bash
# All shards (recommended — runs all pending migrations)
bash backend/scripts/migrate-all-shards.sh

# Dry-run first to preview
bash backend/scripts/migrate-all-shards.sh --dry-run

# Single DB manually (if one shard failed)
DATABASE_URL=$DB_URL_KSA npx node-pg-migrate up \
  --migrations-dir backend/migrations \
  --no-lock

# Verify the enum values exist on a shard
psql $DB_URL_KSA -c "SELECT enumlabel FROM pg_enum JOIN pg_type ON pg_type.oid = enumtypid WHERE typname = 'payment_method' ORDER BY enumlabel;"
```
