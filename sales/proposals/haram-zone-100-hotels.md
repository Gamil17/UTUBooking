## Special Contract Terms — Haram Zone

### Dynamic Hajj / Umrah Pricing
- Contracts must allow **dynamic peak pricing** — no rate caps during Hajj (Dhul Hijjah 8–13) and Ramadan
- Agreed rate basis: **net BAR** (Best Available Rate) minus agreed commission %
- Rate loading obligation: hotels must load rates in UTUBooking CRS **90 days in advance minimum**

### Guaranteed Allocation Blocks
- P1 hotels (★★★★★ within 200m): Negotiate **20–50 room allocation blocks** for Hajj season
- P2 hotels: **10–20 room blocks**
- Allocation release: 30 days before arrival; unreleased rooms return to hotel inventory

### Required Data Fields for Platform Integration
```
hotel_id          UUID (UTUBooking internal)
property_name_en  string
property_name_ar  string
stars             integer 1–5
distance_haram_m  integer (metres from Masjid al-Haram/Nabawi front gate)
haram_view        boolean
zamzam_floor      boolean (Makkah only — floor with Zamzam water service)
qibla_direction   boolean (room Qibla indicators provided)
prayer_times_tv   boolean (prayer times on in-room TV)
halal_certified   boolean
gender_segregated_facilities boolean
wudu_areas        boolean
latitude          decimal
longitude         decimal
check_in_time     string HH:MM
check_out_time    string HH:MM
cancellation_policy enum: free_24h | free_48h | free_7d | non_refundable
```

### Distance Verification Process
1. Hotel self-reports distance from Haram gate
2. UTUBooking validates via Google Maps + Saudi Geospatial Authority coordinates
3. Tolerance: ±50m accepted; discrepancy >50m → renegotiate tier
4. Satellite verification for new/planned properties
