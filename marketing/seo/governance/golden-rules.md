# UTUBooking SEO Content — 5 Golden Rules
**Applies to:** All AI-generated blog posts, travel guides, and landing page copy
**Owner:** Marketing Manager
**Review:** Quarterly (start of each quarter)

---

## Rule 1: Never Publish Without Human Review

Claude drafts; a human editor approves before any CMS upload.

**What this means in practice:**
- Every post goes through the Step 5D Post Review Checklist before upload
- The Notion status must reach "Approved" before CMS access
- No post goes live on a Friday afternoon or public holiday (no safety net)
- If the editor is unavailable, the post waits — quality over cadence

**Who is responsible:** Marketing Manager (final approval authority)

---

## Rule 2: Fact-Check All Statistics

Claude cites sources inline. Click through and verify every single one before publishing.

**What to check:**
- Is the source URL still live? (links die; update or remove dead sources)
- Is the data from the last 12 months? (travel statistics change fast)
- Is the source credible? (acceptable: GASTAT, Saudi Tourism, IATA, UNWTO, official hotel groups)
- Does the statistic actually say what Claude claims it says? (paraphrasing errors happen)

**Red flag sources to verify carefully:**
- Any percentage claims ("X% of pilgrims...")
- Visitor number claims ("X million Umrah pilgrims in [year]")
- Price range claims (verify against current UTUBooking listings)
- Competitor mentions (never state competitor pricing or claims — legal risk)

**If a stat can't be verified:** remove it entirely rather than publish an unverified claim.

## Rule 3: Arabic Review by Native Speaker

All Arabic content generated by Claude must be reviewed by a native Gulf Arabic speaker before publishing.

**Why this matters:**
- Claude generates Modern Standard Arabic (MSA) that can sound unnatural to Gulf Arabic speakers
- Gulf dialect phrasing in the Answer Capsule requires native tone, not machine translation
- Islamic terminology must be used correctly and respectfully — errors here damage brand trust
- RTL formatting, Arabic numerals, and diacritics must be checked

**Review process:**
1. Claude saves Arabic draft with all `[REVIEW]` flags marked
2. Arabic draft sent to native speaker reviewer (internal team or contracted reviewer)
3. Reviewer corrects tone, terminology, and RTL formatting issues
4. Corrected version returned and uploaded — original Claude draft archived

**Turnaround SLA:** 48 hours from receipt

## Rule 5: Monitor and Roll Back if Quality Drops

If blog post quality degrades (wrong tone, missing capsules, bad sources), investigate before continuing.

**Symptoms of quality degradation:**
- Posts missing the TL;DR block
- Answer Capsules longer than 80 words or absent entirely
- Em dashes appearing in the text
- Source citations absent or unverifiable
- Internal links forced or missing
- Arabic posts not flagged with `[REVIEW]` markers

**Diagnosis steps:**
```bash
# Check what changed in the Skill recently
git log --oneline -10 skills/seo-copywriter/SKILL.md

# Compare current Skill to last known-good version
git diff [last-good-commit] skills/seo-copywriter/SKILL.md

# Roll back Skill if needed
git checkout [last-good-commit] -- skills/seo-copywriter/SKILL.md
```

**Escalation:** If quality issues persist after rolling back SKILL.md, raise with the Ops team and open a task to audit the full skills/seo-copywriter/ folder.
